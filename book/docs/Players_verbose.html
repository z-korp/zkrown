CARD ( card_id, name, #player_id )
  • Le champ card_id constitue la clé primaire de la table. C'était déjà un identifiant de l'entité CARD.
  • Le champ name était déjà un simple attribut de l'entité CARD.
  • Le champ player_id est une clé étrangère. Il a migré par l'association de dépendance fonctionnelle HAS à partir de l'entité PLAYER en perdant son caractère identifiant.

SUPPLIES ( #unit_id, #card_id )

  • Le champ card_id fait partie de la clé primaire de la table. C'est une clé étrangère qui a migré directement à partir de l'entité CARD.
